# 01 | 什么是 MCP？

## 🎯 学习目标

- 理解 MCP（Model Context Protocol）是什么
- 了解 MCP 解决什么问题
- 掌握 MCP 的核心定位：AI 世界的"USB-C 接口"

---

## 📖 概念讲解

### MCP 是什么？

**MCP（Model Context Protocol，模型上下文协议）** 是一种开放标准协议，用于规范 AI 模型与外部数据源、工具之间的通信方式。

> **简单理解**：MCP 是 AI 领域的"USB-C 接口"——无论什么品牌的设备，只要符合 USB-C 标准，就能即插即用。

### 为什么会诞生 MCP？

在 MCP 出现之前，AI 模型想访问外部世界是这样的：

```
AI 想读文件？→ 写一个文件读取插件
AI 想查数据库？→ 写一个数据库插件
AI 想调 API？→ 写一个 API 插件
AI 想用搜索？→ 再写一个搜索插件
...
```

每个插件都是独立的，没有统一标准：
- 数据格式不统一
- 接口协议不统一
- 无法复用，换个 AI 就得重写

MCP 出现后：

```
AI ──── MCP ──── 文件系统 Server
              ──── 数据库 Server
              ──── API Server
              ──── 搜索 Server
```

**一次开发，处处运行**。只要符合 MCP 协议，任何 AI 都能用这些 Server。

### MCP 的三个核心原语

MCP Server 向 AI 暴露三种能力：

| 原语 | 作用 | 类比 |
|------|------|------|
| **Tool（工具）** | AI 可以主动调用的函数（有副作用） | Agent 的"手"——能改变世界 |
| **Resource（资源）** | AI 可以读取的数据（只读） | Agent 的"眼睛"——能获取信息 |
| **Prompt（提示模板）** | 预定义的提示词，带参数 | Agent 的"快捷指令"——一键执行复杂任务 |

## 💡 生活中的类比

把 MCP 放在"餐厅厨房"系列里来理解：

**主厨**（Agent）需要做菜，但每换一家厨房，锅碗瓢盆的操作方式都不同：
- 这口锅是旋转拧开的，那口锅是卡扣式
- 这台烤箱用旋钮，那台烤箱用触屏

**没有 MCP 的时代：**
> 主厨每换一家厨房，得重新学一遍所有设备的操作方式。

**有 MCP 的时代：**
> 所有厨房设备统一接口（USB-C）。主厨学会一种操作方式，所有厨房的设备都能用。

**MCP 就是厨房里的 USB-C 接口标准。** 主厨（Agent）只需要学会一种连接方式，所有厨房电器都能即插即用。

---

## 📝 章节回顾

**记住这三个关键点：**

1. **MCP = USB-C for AI**
   统一标准，一次开发，到处运行

2. **MCP 解决的是"连接"问题**
   AI 与外部世界之间的标准化通信协议

3. **三个核心原语：Tool / Resource / Prompt**
   - Tool = 能做的事（有副作用）
   - Resource = 能读的数据（只读）
   - Prompt = 预定义的快捷指令

**MCP 核心公式：**

```
MCP = Host（运行 AI 的应用）+ Client（Host 内部的协议处理器）+ Server（提供工具/资源的服务）
```
