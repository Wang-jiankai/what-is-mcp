# 03 | 参考答案：理解 JSON-RPC 协议

## 📋 基础任务参考答案

### 消息 1

```json
{
  "jsonrpc": "2.0",       // JSON-RPC 版本
  "id": 42,               // 请求 ID（用于匹配响应）
  "method": "tools/list", // MCP 方法（列出所有可用工具）
  "params": {}            // 参数（空）
}
```

- **类型**：请求（Request）
- **method**：`tools/list` — 告诉 Server 列出所有可用工具
- **result**：无（是请求，不是响应）
- **id 字段作用**：请求的唯一标识，响应中必须包含相同的 id，以便 Host 匹配请求和响应

### 消息 2

```json
{
  "jsonrpc": "2.0",       // JSON-RPC 版本
  "id": 42,               // 与请求的 id 匹配
  "result": {             // 执行结果
    "tools": [
      { "name": "readFile", "description": "读取文件" }
    ]
  }
}
```

- **类型**：成功响应（Success Response）
- **method**：无（响应中没有 method）
- **result**：Server 返回的工具列表
- **id 字段作用**：与消息 1 的 id=42 匹配，证明这是对"列出工具"请求的响应

### 消息 3

```json
{
  "jsonrpc": "2.0",       // JSON-RPC 版本
  "id": 99,               // 与原请求的 id 匹配
  "error": {              // 错误信息
    "code": -32602,       // 错误码
    "message": "参数错误：phone 必须是 11 位数字"
  }
}
```

- **类型**：错误响应（Error Response）
- **method**：无
- **error**：错误信息（不是 result）
- **id 字段作用**：与出错请求的 id 匹配

## 📋 进阶任务参考答案

### 传输方式对比图

```
=== stdio 方式（本地）===
┌─────────────────────────────────────┐
│           Claude Desktop (Host)      │
│  ┌─────────┐  ┌─────────┐          │
│  │ Client 1│  │ Client 2│  ...     │
│  └───┬─────┘  └───┬─────┘          │
└──────┼────────────┼────────────────┘
       │  stdin/    │  stdout/
       ▼            ▼
  ┌─────────┐  ┌─────────┐
  │ Server 1│  │ Server 2│  (子进程)
  └─────────┘  └─────────┘

特点：Server 作为 Host 的子进程启动，通过管道通信


=== HTTP 方式（远程）===
┌─────────────────────────────────────┐
│           Claude Desktop (Host)      │
│  ┌─────────┐  ┌─────────┐          │
│  │ Client 1│  │ Client 2│  ...     │
│  └───┬─────┘  └───┬─────┘          │
└──────┼────────────┼────────────────┘
       │ HTTP POST │ HTTP Response
       ▼            ▼
  ┌─────────────────────────┐
  │     远程 HTTP Server     │
  │  (可以是任何地方部署的)    │
  └─────────────────────────┘

特点：Server 部署在远程，通过 HTTP 协议通信
```

### 适用场景

| 传输方式 | 适用场景 |
|---------|---------|
| **stdio** | 本地 Server、需要访问本地文件的场景 |
| **HTTP** | 远程 Server、需要跨网络访问的场景 |
